# ProxySwitch

Небольшое desktop-приложение (Python + CustomTkinter) для быстрого переключения системного прокси между профилями.

Основная идея: ты создаешь несколько профилей прокси (HTTP / HTTPS / SOCKS5), выбираешь активный, и `ProxySwitch` ставит системный прокси на локальный `127.0.0.1:18080`, где крутится встроенный routing-прокси.  
Он уже решает, что отправлять через upstream-прокси, а что пускать напрямую по правилам.

## Что умеет

- Профили прокси с именем, типом, хостом, портом
- Авторизация (логин/пароль) для upstream-прокси
- Быстрый тест доступности профиля перед использованием
- Правила маршрутизации:
  - `*.corp.local`
  - `192.168.*.*`
  - `10.0.0.0/8`
- Применение правил сверху вниз (первое совпадение побеждает)
- Включение/отключение системного прокси из UI
- Логи в `~/.proxyswitch/debug.log`

## Как это работает под капотом

1. При активации профиля поднимается локальный proxy listener на `127.0.0.1:18080`.
2. Система перенаправляет HTTP/HTTPS на этот локальный адрес.
3. Для каждого запроса:
   - если правило говорит `direct`, идет прямое подключение к цели;
   - иначе идет CONNECT через выбранный upstream-прокси.
4. При отключении профиля локальный listener останавливается и системный прокси сбрасывается.

## Требования

- Python 3.10+ (рекомендуется 3.11/3.12)
- macOS / Windows  
  (Linux поддерживается только через переменные окружения, без системной интеграции уровня network manager)
- Для SOCKS5 нужен `PySocks` (уже в `requirements.txt`)

## Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m proxyswitch
```

## Сборка приложения (macOS `.app`)

Проект уже настроен под PyInstaller через `ProxySwitch.spec`.

```bash
source .venv/bin/activate
pip install -r requirements.txt
pip install pyinstaller
pyinstaller ProxySwitch.spec
```

Готовый `.app` появится в `dist/ProxySwitch.app`.

## Где хранятся данные

- Профили: `~/.proxyswitch/profiles.json`
- Лог: `~/.proxyswitch/debug.log`

## Важные нюансы

- На macOS используется `networksetup`, поэтому может потребоваться подтверждение прав системы.
- Правила применяются к `host` (а не к полному URL).
- Если правило не совпало, по умолчанию используется upstream-прокси.

## Быстрый чек после запуска

1. Добавь профиль.
2. Нажми `Проверить` — должен быть статус `ok`.
3. Нажми `Применить`.
4. Проверь, что в статусбаре видно `routing 127.0.0.1:18080`.
5. Нажми `Отключить` и убедись, что системный прокси сброшен.
